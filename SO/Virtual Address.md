# Virtual Address

> [!abstract] Definition
> A **virtual address** is the address generated by a [[Process|user program]] — an offset into its private [[Address Space]]. It is *not* a physical location in DRAM. The OS, with hardware help ([[MMU]]), translates each virtual address to a real **physical address** before any actual memory access happens.

## The Single Most Important Sentence in Memory Virtualization

> [!example] Memorize this
> *"Every address generated by a user program is a virtual address."*
>
> — [[Ch 12 — A Dialogue on Memory Virtualization|Ch 12]], the Professor's parting line.

When user code writes `*p = 7`, computes `&x`, executes `call foo`, the addresses involved are *virtual*. The hardware silently translates each one on the fly.

## Virtual vs Physical

| Virtual | Physical |
|---|---|
| What the [[Process]] sees | What DRAM actually uses |
| Per-process; private | Shared by all processes + kernel |
| Zero-based, large, contiguous | Fragmented, bounded by RAM size |
| Generated by every load/store/jump | Used to drive memory bus |
| Translated by [[MMU]] | Final destination |

## Quick Demo

```c
int x = 7;
printf("%p\n", &x);   // prints, e.g., 0x7ffe...

// run twice, in two terminals — both probably print
// the same virtual address, yet they don't see each
// other's value of x. Different physical addresses
// behind the same virtual one.
```

That divergence *is* the illusion. See [[Address Space]] for the full picture.

## Why Virtual Addresses Exist

- **[[Isolation (OS)|Isolation]]** — A's address `0x1000` and B's address `0x1000` map to *different* physical bytes. A can't peek at B.
- **Convenience** — every program can pretend it owns memory starting at zero. No coordination, no fixed loader-time addresses.
- **Flexibility** — the OS can swap pages out to disk, share pages between processes, or relocate them, all without the program noticing.

## Translation: Preview

The mechanisms that actually convert virtual → physical evolve through Ch 15–20:

1. [[Base and Bounds]] (Ch 15) — add a base register, check against a bound.
2. [[Segmentation]] (Ch 16) — multiple base/bound pairs per address space.
3. [[Paging]] (Ch 18) — break virtual + physical memory into fixed-size chunks; map via a [[Page Table]].
4. [[TLB]] (Ch 19) — cache recent translations to make paging fast.
5. [[Multi-Level Page Table]] (Ch 20) — shrink page tables themselves by paginating them.

> [!tip] The address bus is **never** virtual
> By the time an address reaches the memory controller / DRAM, it has been translated. DRAM sees only physical addresses. Virtual addresses live entirely inside the CPU pipeline + [[MMU]].

## Related Notes

- [[Address Space]] — the container of all of a process's virtual addresses.

- [[TLB]] — the translation cache.
